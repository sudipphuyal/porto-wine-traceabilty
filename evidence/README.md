# Evidence

Do not place fabricated screenshots or fake logs here.

After DAML and Canton are installed, capture reproducible evidence with:

```bash
./scripts/capture-evidence.sh
```

A valid evidence run should include:

- UTC timestamp generated by the machine.
- DAML/Canton versions.
- DAR build output.
- Daml Script result for the end-to-end scenario.
- Recall and negative-test results.
- If Canton is used as the backing ledger, the real participant transaction/event identifier reported by the installed tooling.

If the Canton Domain / Synchronizer is unavailable during custody transfer finalisation, the transaction must not be treated as final. Retry logic must be idempotent from the client side and must not create duplicate custody events.
